# Customer Loyalty / Churn Decision Tree Model
## Purpose
I.	Understand a modeling problem in the context of a business situation. <br>
II.	Design and build a custom modeling pipeline to solve that problem. <br>
III.	Fine tune the pipeline for optimal performance on an evaluation metric. <br>
## Challenges
I.	Implementing the column transformer as an argument in the grid search method when it was not covered completely in class. <br>
II.	Getting the data in both one-hot encoded and numerical form for different analyses, including generating an ROC curve. <br>
III.	Being mindful that the target data was imbalanced, necessitating an F1 test for model evaluation.
## Learnings
I.	Looking at the feature data beforehand would have revealed what insight could have come from an analysis. <br>
II.	Reading and following the challenge criteria more carefully would have resulted in a better analysis and results. <br>
